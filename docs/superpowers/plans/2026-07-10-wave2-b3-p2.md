# P2 · Tiered Book X-Ray — Implementation Plan

> **For track implementers:** execute this plan task by task in the assigned worktree.
> Use red-green-refactor for each task, commit only the declared files, and stop on a
> conflict with section 8 rather than widening scope.

**Goal:** Ship the P2 static-fusion exhibit: one synthetic 15-manager book-level net-beta
posterior, an honest 90% interval, manager-level uncertainty provenance, and a measured
information-gain comparison across all-R, actual-mix, and all-at-least-E transparency.

**Architecture:** A pure `flagships/book_xray/fusion.py` module implements the normal-normal
measurement update, independent-manager book aggregation, variance provenance, and the
static information-gain gate. `demo_data/p2_xray.py` builds one shared synthetic market,
emits each manager at its assigned transparency tier, uses the existing S2 regression for
the demo's returns-only observation, opts into the substrate's E-tier coarsening, and writes
deterministic JSON. The page renders committed JSON only. The temporal filter and the X1
exposure-error atlas rows remain wave-3 work under binding section 8 ruling 5.

**Tech stack:** Python 3.11, numpy, pandas, existing simulator and Jinja site builder,
vanilla JavaScript for value-to-pixel mapping and precomputed-state switching only.
