section 8 ruling 1 and the section 6.5 kill rule, the active layer is therefore **hybrid search
plus extraction tables/brief, no production graph expansion**. The graph candidate may render
only in the evaluation comparison, labelled `candidate — gate not cleared`. This card may still
go live in its specified fallback state; a larger planted eval is the wave-3 route to re-earn it.

**Interfaces:**

```python

```

Gate state is `pass`, `fail`, or `insufficient`. Pass requires mean paired recall@10 uplift
at least 0.10 **and** interval lower bound above zero. Fewer than two paired queries is
`insufficient`; failure/insufficiency selects `hybrid_search`.

- [ ] Pin illustrative top-3 recall/precision 0.67→1.00.
- [ ] Pin formal current recall@10 1.00→1.00 and state `insufficient`, never pass.
- [ ] Use a synthetic multi-query unit fixture to prove pass requires both threshold and
  interval; this fixture tests gate logic only and is not emitted as E3 evidence.
- [ ] Wilson/bootstrap intervals are bounded and deterministic. Current binary relevance
  produces binary nDCG; make no graded-relevance claim.
- [ ] Run eval tests; commit `feat(e3): paired retrieval gate and honest hybrid fallback`.
